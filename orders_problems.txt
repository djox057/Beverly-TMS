3. Load LOCKED orders from cache:
   a. Check local IndexedDB for cached data
   b. Compare local cache version vs server archive_version
   c. If versions match: use local cache (zero network cost)
   d. If versions differ: fetch from Supabase Storage (archived-orders/locked-orders.json)

4. CRITICAL STEP - Fetch missing locked orders from database:
   ```javascript

   while (true) {
     const { data: batch } = await supabase
       .from("orders")
       .select("*")  // <-- ONLY basic fields, NO JOINS!
       .eq("locked", true)
       .order("updated_at", { ascending: false })
       .range(offset, offset + batchSize - 1);
     
     // ... merge with archived data
   }
   ```
   This ensures recently-locked orders appear even if not in archive.

5. Enrich locked orders with lookup data:
   - Batch fetch trucks, trailers, drivers, brokers, companies
   - Fetch pickup_drops, order_files, order_transfers from database

6. Deduplicate: Remove locked orders if an unlocked version exists
7. Merge: Combine unlocked (100 max) + all locked orders
8. Transform: Convert to camelCase UI shape via transformOrders()
